# HMIS Roadmap

This roadmap tracks the staged build-out of the Human Mind Intelligence System.

## v1 — Foundation

**Goal**: establish the complete research foundation for HMIS.

### Deliverables
- [x] Positioning README with doctrine and guardrail
- [x] Agent and Hermes specifications
- [x] Machine-readable system manifest
- [x] Model layer directories and doctrine-aligned READMEs
- [x] Schema contracts for signals, mind-state, and cognitive profile
- [x] Skill contracts for empathy, reasoning, and memory
- [x] Workflow definitions for onboarding, inference, and evaluation
- [x] Issue template and codex task board

### Acceptance criteria
- A new contributor can understand the repo structure quickly.
- Hermes' operating contract is explicit and non-clinical.
- All schemas parse and conform to draft-07 structure.
- The foundation is ready for implementation work, but remains research-only.

## v2 — Implementation

**Goal**: build the first working inference pipeline.

### Planned work
- [ ] Hermes inference engine
- [ ] Schema validation automation
- [ ] Skill implementations
- [ ] Test suite and CI workflows
- [ ] Calibration data and evaluation fixtures

## v3 — Adaptation

**Goal**: support longer-horizon learning while preserving doctrine and privacy.

### Planned work
- [ ] Consent-aware profile persistence
- [ ] Evaluation dashboards and calibration loops
- [ ] Multi-session adaptation rules
- [ ] Tribe-aware inference refinement
- [ ] SDK or package surface for downstream systems

## Guiding principles

1. Human-first, never clinical.
2. Evidence before interpretation.
3. Modularity across models, skills, and workflows.
4. Research rigor before productionization.
